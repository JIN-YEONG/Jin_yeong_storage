과적합
    -> 모델이 훈련을 하는 과정에서 train_data값에 최적화 되어 있는 상태
    -> train_data에 대한 정확도는 높지만 새로운 데이터에 대한 정확도는 낮다


validation_data
    -> 검증 데이터
    -> 검증 데이터가 필요한 이유는 훈련도중 머신이 훈련결과를 검증데이터와 비교하여 
        과적합을 방지하는 역할을 한다.

데이터 슬라이싱
    -> 큰 데이터를 작은 데이터로 나누는 작업
        큰 데이터에서 train, test, val 데이터로 나눈때 사용

train_test_split
    -> sklearn만에 있는 데이터 슬라이싱 함수
    -> 정해진 size만큼 데이를 자르고 랜덤하게 섞어준다.

    x_train,x_test, y_train, y_test = train_test_split(
        x, y, test_size= 0.4   # 40%
    )
    # train_size를 넣어 train_data의 크기를 지정할 수도 있음
    # 실수를 넣으면 백분률, 정수를 넣으면 개수로 자른다.

데이터 슬라이싱에 랜덤이 필요한 이유
    -> 데이터에 의도치 않은 페턴이 존재하는 경우 그로 인데 
        원하지 않는 결과가 출력되는 것을 방지하기 위해 
        데이터를 섞는다.

np.transpose()
    ->배열의 shape를 바꿔준다.
        ex) (1,2) -> (2,1)
    -> 3차원 이상의 배열의 경우
        np.transpose(x, [1,2,0])의 형식으로 
        (5,3,4) 로 지정하여 원하는 shape로 변경할 수 있다.
